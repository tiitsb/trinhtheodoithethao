Hệ Thống Sửa Lỗi Mã Nguồn (Debug)

1. Nguyên văn đoạn mã lỗi (Từ đề bài)
function togglePass() {
    let passField = document.getElementById("password");
    if (passField.type = "password") {
        passField.type = "text";
    } else {
        passField.type = "password";
    }
}
2. Đoạn mã đã sửa lỗi hoàn chỉnh & Giải thích
function togglePass() {
    let passField = document.getElementById("password");
    
    // SỬA LỖI TẠI ĐÂY: Thay đổi toán tử gán (=) thành toán tử so sánh bằng (===)
    // Lỗi cũ: if (passField.type = "password") khiến thuộc tính luôn bị gán lại thành "password" và trả về giá trị đúng (truthy), dẫn đến nhánh 'else' không bao giờ được chạy.
    if (passField.type === "password") { 
        passField.type = "text";
    } else {
        passField.type = "password";
    }
}
Chi tiết các lỗi sai được tìm thấy:
  • Lỗi Logic Toán Tử (Nghiêm trọng): Tại câu lệnh điều kiện if (passField.type = "password"), lập trình viên đã dùng nhầm một dấu bằng = (đây là phép toán gán giá trị). Do đó, câu lệnh này không kiểm tra xem trường dữ liệu có phải mật khẩu không, mà nó ép định dạng trường đó thành password rồi luôn luôn trả về kết quả đúng, khiến tính năng ẩn mật khẩu bị tê liệt hoàn toàn.
    ➔ Biện pháp xử lý: Thay thế bằng toán tử so sánh nghiêm ngặt === (hoặc ==).
